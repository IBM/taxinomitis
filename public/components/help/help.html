Teacher / Admin UI issues

Number of available ML models in the Teacher's page is incorrect

The Teacher's page shows the number of machine learning models available to your class. (This page is only accessible to teachers or group leaders). This value is calculated by taking the number of API keys / credentials that the teacher has created, and multiplying it by the number of machine learning models that can be created with a single API key.

Unfortunately, this can be misleading. The value is based on assumptions about which Pricing Plan were used when creating API keys.

For text projects, the assumption is that "Standard Plan" will be used for creating Watson Assistant (formerly Watson Conversation) credentials, as this supports 20 "workspaces". The teacher's page multiplies the number of Watson Assistant credentials by 20 when displaying

"Your class is allowed to have up to ?? machine learning models in total across all their text projects."

The "Lite Plan" supports only 5 "workspaces", so if any of the Watson Assistant credentials were created in the Lite pricing plan, then this means the actual limit will be less than displayed on the Teacher's page.

For images projects, the assumption is that "Lite Plan" will be used for creating Watson Visual Recognition API keys, as this supports 1 "custom model". The teacher's page multiplies the number of Watson Visual Recognition API keys by 1 when displaying

"Your class is allowed to have up to ?? machine learning models in total across all their images projects."

The "Standard Plan" doesn't have a limit on the number of custom models, so if any of the Watson Visual Recognition API keys were created with the Standard pricing plan, this means there isn't actually a limit on the number of machine learning models for images projects, despite what is shown on the Teacher's page.

Sorry for any confusion that this page causes. I will be updating the Teacher's admin page so it asks for the Pricing Plan for each set of credentials or API keys, so that I can do the appropriate multiplication in the UI.

In the meantime, the correct limit is whatever is shown on IBM Cloud where the credentials are created. More detailed information about all of this can be found in this PDF document about using Watson API keys with Machine Learning for Kids.

If you're not sure how to work out the limit for your class, please let me know and I'll try and help.

Missing student account from the list in the teacher's page

User management for Machine Learning for Kids is implemented using the third party service, Auth0.

Occassionally, the list of students that Auth0 returns can get a little out of date, and fail to reflect students that have recently been created (or similarly, still show students that have been recently deleted).

In these instances, the students have normally been successfully created (or deleted) and it is just that the list of students being displayed doesn't reflect recent updates.

Auth0 will normally report such incidents on their status page as user indexing delays or problems with their Search API.

The best thing to do in these cases is to wait for the Auth0 indexing to catch up.

What is Assistant? What happened to Conversation?

In March 2018, the "Watson Conversation" service was renamed to "Watson Assistant". Any previous references to "Conversation" can be read as "Assistant" now. Any Watson Conversation service instances or credentials created prior to this point can continue to be used as equivalent to Watson Assistant.

Why are machine learning models automatically deleted?

This was a feature requested by schools when Machine Learning for Kids was first developed. It came from the fact that there is a limit for how many machine learning models a class can have at any given time, and a need to make it easy to manage this limit.

Rather than requiring kids to have to clean up after a lesson and delete their models (in order to allow the next class to be able to create models), the tool can automatically clean up by deleting the models after the lesson is over. This makes it easy to run the next lesson, and the next lesson after that, without manual clean-ups in between.

The time limit before models are deleted is configurable. In a school, where lessons are 1 or 2 hours long, deleting models after two hours might be sensible. For a weekly coding club, six days might be a better fit to make sure that models are cleared up before each new club session.

To check the time limit for your class, visit the Restrictions section of your Teacher page.

To change the time limit for your class, please send me an email.

Note: Only machine learning models are automatically deleted, not the training data that was used to create them. So if a model is automatically deleted for a project that a student is still working on, they can always just click the "Train" button again to create the model again.

Account issues

What is the difference between creating accounts or using "Try it now"?

Using "Try it now" to means there is no need to log on or create an account. There are no passwords for students to have to remember, and no need for teachers to have to provide personal information. It also saves teachers the time needed to manage student accounts.

However, creating an account does enable additional features. Students can save their work and come back to it in a later lesson. They can work on a project together with other students in their class. They can log on from home and show their family what they've made.

I've created a class account but my teacher's logon isn't working

You need to verify your email address before you'll be able to log on.

When you created the class account, an email will be sent to the email address you provided, with a link to click on. If you click on that link, your email address will be verified and the class account will be activated. You should be able to log on now.

If you haven't seen the verify email yet, make sure it's not gone into your spam folder.

If you still can't find it, let me know and I'll try and help.

Deleting accounts

Student accounts

Students cannot delete their own accounts.

A teacher can delete a student account at any time by visiting the "Teacher" page and clicking the "Delete student account" button next to the student account you want to delete.

This will delete all information stored about that student and all of their projects and machine learning models.

It cannot be undone.

Teacher accounts

Deleting a teacher account will delete the entire class and all of the student accounts in it. It cannot be undone.

This cannot be done from the UI (although I'm working on adding it).

To delete a teacher account in the meantime, please email me and I will do it for you.

What information is stored about users?

Teacher accounts

I store the username and email address that you provide when you create the account, so that I can keep in touch with you.

Student accounts

I store the usernames for student accounts, so that I can let them log in. I have no email addresses or contact details for students. I have no other personally identifying information about students.

I also recommend that students are given generic usernames (e.g. "student1") so that students are not identifiable outside of their school or coding group.

General information stored about all accounts

User management for Machine Learning for Kids is implemented using the third party service, Auth0. They store the IP address that you last logged into Machine Learning for Kids from, and the type of browser you used. I've never found a reason to use that, but it is stored if I did want to go and look for it.

Errors that happen in the web browser are captured using the third party service, Sentry. If something goes wrong, it will capture information about the error, including your username, IP address, type of browser you were using, and a technical description of what went wrong.

I use Google Analytics so that I know how many users visit Machine Learning for Kids each day. Although it captures information such as geographic location and browser type, this is only ever displayed to me in an anonymised aggregate way.
