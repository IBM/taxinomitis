The type of machine learning model you trained in this project is called a neural network.
The next few pages will explain how your model was created.
These diagrams aren't an exact description of your model. Your model is larger and more complicated, but these diagrams are easier to explain.
The model is made of nodes, arranged in vertical layers.
Models with a lot of layers are described as deep.
Deep models can give better results, but take longer to train.
Let's start with one of the training examples used to create your model.
Your training example is shown as input on the left.
To use your training example to train the model, first the text needs to be turned into a series of numbers.
There are lots of ways to do this.
If you build your own machine learning model, you need to choose a way that works well for your particular project.
One way of turning your text into numbers is to make each number a count of how many times a word is included.
There are other things you can count. Things that are helpful to count depend on the project you are doing, and what you want your model to learn to recognize patterns in.
Counting things is quick and simple, but not as good for projects where the order of words is important.
If you build your own machine learning model, you need to choose the method for turning text into numbers.
Your model used a more complicated technique that does include the order of words in the text.
The numbers from the input layer go into sums that calculate values for the nodes in the next layer.
The lines on the diagram show which input values are used to calculate which values in the next layer.
There are lots of different types of calculation you can use. Not every node in the model has to use the same calculation.
If you build your own machine learning model, you need to choose which equations to use for each node.
The functions to use for each node in your model were chosen for you.
Each line in the diagram has a weight number that can be used in the sum.
Training the model is how we work out the best number to put on each line, as you'll see in a moment.
Each node in the diagram has a bias number that is used in the sum as well.
Training the model is also how we work out the right bias number, as you'll see in a moment.

One run-through of every training example you collected through the model is called an epoch.
Each time through, the weights for each node are nudged a little more, until hopefully the error rate for predictions are as low as they can be.
If you build your own machine learning model, you need to choose how many epochs you want to run.
